# 轻量化提醒应用项目文档

更新时间：2026-07-16

## 1. 项目定位

轻量提醒是一个 Windows 桌面端轻量任务捕捉和提醒应用。它的核心目标是让用户在不切换复杂工具的情况下，把聊天、会议、邮件、文档、网页里的零碎待办快速收集到一个常驻桌面的任务面板里，并在需要时收到提醒。

本项目不定位为完整项目管理系统，不承载多人协作、权限审批、甘特图、看板工作流、云同步等重型能力。它更像一个“桌面边栏便签 + 快速收集器 + 本地提醒器”。

## 2. 当前版本状态

当前版本为 `0.1.0`，已经具备可运行的 Electron 桌面应用形态。

已实现能力：

- 桌面浮窗任务面板
- 紧凑任务列表，详细操作按需展开
- 托盘显示 / 隐藏窗口
- 全局快捷键
- 本地 JSON 存储任务、分组、设置
- 任务新增、编辑、完成、删除、置顶
- 任务分组、切换、重命名、删除
- 删除分组时任务自动移回收集箱
- PM 状态：待处理、等反馈、风险
- 任务提醒时间编辑、清除、逾期筛选
- 到期系统通知、提示音和应用内提醒面板
- 精确定时、30 秒兜底扫描、系统唤醒后补查
- 设置页测试提醒和 Windows 通知设置入口
- 到期任务窗口唤起、聚焦和任务栏闪烁
- 延后提醒：+10、+30、+1h、明天
- 从剪贴板文本创建任务
- 可选自动复制当前选中文字并恢复剪贴板
- 从文件或剪贴板创建图片任务
- 给已有任务追加图片附件
- 设置页：开机启动、置顶、透明度、快捷键、剪贴板行为
- 顶部任务搜索：在当前分组或侧边栏筛选范围内即时检索
- 收藏、标记置顶与待办 / 等反馈 / 风险状态切换
- 任务拖动到分组；侧边栏支持待办、逾期、等反馈、风险、收藏、标记、全部、已完成和归档视图
- 紧凑任务卡与底部横向操作栏；有图片时展示一张缩略图
- 已完成任务归档与恢复；归档任务不会出现在普通任务、全部和已完成视图
- 数据备份与恢复，包含任务、分组、设置和图片附件；恢复前自动保留恢复前备份
- 任务导出：CSV、Markdown，均包含归档任务
- 打包 Windows portable exe 和文件夹版

暂未建立：

- 完整的端到端自动化测试
- lint / formatter
- TypeScript 类型约束
- 数据迁移版本号
- 多窗口或多设备同步

## 3. 技术栈

实际技术栈：

- Electron `^31.7.7`
- electron-builder `^26.15.3`
- Node.js CommonJS
- 原生 HTML/CSS/JavaScript
- 本地 JSON 文件存储

注意：项目没有使用 React、Vue、Vite、Webpack 或 TypeScript。后续修改应优先沿用当前结构，除非明确要做一次架构升级。

说明：

- `src/main/main.js` 是主进程核心文件，包含窗口、托盘、快捷键、数据读写、提醒调度、图片附件和 IPC。
- `src/main/reminderParser.js` 负责中文自然语言提醒时间解析，可脱离 Electron 测试。

- `build/` 保存打包图标。
- `release/` 是打包输出目录，不是源码。
- `node_modules/` 是依赖目录，不是源码。
- `test/reminderParser.test.js` 使用 Node.js 内置测试框架验证时间解析。

## 6. 主进程职责

文件：[src/main/main.js](src/main/main.js)

主进程当前承担较多职责：

- 应用单实例锁
- 固定 `userData` 目录
- 日志写入
- 主窗口创建和生命周期
- 托盘图标和菜单
- 全局快捷键注册
- 设置读取、写入、应用
- 任务读取、写入、创建、更新、删除
- 分组读取、写入、创建、重命名、删除
- 剪贴板文本读取
- 选中文字自动复制
- 图片附件复制和保存
- 系统通知
- 提醒定时检查
- IPC handler 注册

关键常量：

- 默认分组：`DEFAULT_GROUP_ID = 'inbox'`
- 默认窗口宽度：`390`
- 最小窗口：`340 x 460`
- 折叠高度：`46`
- 提醒兜底检查间隔：`30000ms`

后续如果功能继续增加，建议逐步拆分主进程模块，例如：

- `taskStore.js`
- `groupStore.js`
- `settingsStore.js`
- `windowManager.js`
- `shortcutManager.js`
- `reminderScheduler.js`
- `attachmentStore.js`

拆分时应保持 IPC 名称和数据结构兼容，避免一次性重构引入行为变化。

- `renderGroups()`

主要交互：

- 新增任务
- 切换筛选

- 清除提醒时间
- 移动任务分组
- 切换 PM 状态
- 追加图片
- 删除任务
- 分组新增、重命名、删除

当前渲染方式是手写 DOM，不使用虚拟 DOM。修改时要注意：

## 8. Preload API

文件：[src/main/preload.js](src/main/preload.js)

`preload.js` 暴露 `window.reminderApi`，当前 API 包括：

任务：

- `listTasks()`
- `createTask(input)`
- `updateTask(id, patch)`
- `deleteTask(id)`
- `clearCompleted(groupId)`

剪贴板：

- `createTaskFromClipboard(groupId)`
- `createTaskFromSelection(groupId)`

分组：

- `listGroups()`
- `createGroup(name)`
- `renameGroup(id, name)`
- `deleteGroup(id)`
- `setActiveGroup(id)`

图片：

- `createImageTasksFromFiles(groupId)`
- `createImageTaskFromClipboard(groupId)`
- `attachImagesToTask(taskId)`
- `openAttachment(attachmentPath)`

窗口：

- `rendererReady()`
- `hideWindow()`
- `toggleCollapsed()`
- `toggleAlwaysOnTop(value)`

设置：

- `getSettings()`
- `updateSettings(patch)`

事件订阅：

- `onTasksChanged(callback)`
- `onGroupsChanged(callback)`
- `onSettingsChanged(callback)`
- `onFocusQuickInput(callback)`
- `onHighlightTask(callback)`
- `onReminderDue(callback)`
- `onCollapsedChanged(callback)`

维护原则：

- 新增主进程能力时，先在主进程注册 `ipcMain.handle` 或 `webContents.send`，再通过 preload 暴露最小 API。
- 不要把 `ipcRenderer` 直接暴露给页面。
- API 命名保持领域语义，不暴露实现细节。

### 9.1 任务结构

当前任务结构：

```json
{
  "id": "task_1710000000000_abcd",
  "title": "明天10点确认报价",
  "note": "",
  "sourceText": "",
  "sourceApp": "",
  "status": "todo",
  "priority": "normal",
  "pmStatus": "normal",
  "pinned": false,
  "flagged": false,
  "archivedAt": null,
  "groupId": "inbox",
  "attachments": [],
  "remindAt": "2026-06-30T02:00:00.000Z",
  "createdAt": "2026-06-29T10:00:00.000Z",
  "updatedAt": "2026-06-29T10:00:00.000Z",
  "completedAt": null,
  "notifiedAt": null
}
```

字段说明：

- `status`: `todo` 或 `done`
- `pmStatus`: `normal`、`waiting`、`risk`
- `pinned`: 是否收藏
- `flagged`: 是否标记并置顶
- `archivedAt`: 归档时间；为空表示未归档。归档任务仍保留在本地数据中，可恢复到已完成视图
- `priority`: 当前保留字段，UI 暂未使用
- `sourceText`: 从剪贴板或选区创建任务时保存来源文本
- `sourceApp`: 当前保留字段，暂未自动识别来源应用
- `remindAt`: ISO 时间字符串，为空表示无提醒
- `notifiedAt`: 提醒触发时间，主要用于记录
- `attachments`: 图片附件数组

### 9.2 分组结构

```json
{
  "id": "inbox",
  "name": "收集箱",
  "createdAt": "2026-06-29T10:00:00.000Z",
  "updatedAt": "2026-06-29T10:00:00.000Z"
}
```

规则：

- 默认分组固定为 `inbox`。
- 如果 `groups.json` 缺少 `inbox`，启动时会自动补回。
- 删除非默认分组时，该分组任务会移回 `inbox`。

### 9.3 设置结构

```json
{
  "alwaysOnTop": false,
  "openAtLogin": false,
  "opacity": 1,
  "autoCopySelection": false,
  "restoreClipboard": true,
  "shortcuts": {
    "quickInput": "Control+Alt+Space",
    "clipboardTask": "Control+Alt+T",
    "toggleWindow": "Control+Alt+R"
  }
}
```

规则：

- `opacity` 会限制在 `0.45` 到 `1`。
- `restoreClipboard` 默认 `true`。
- 快捷键为空时会回退到默认值。

### 9.4 图片附件结构

```json
{
  "id": "image_1710000000000_abcd",
  "type": "image",
  "name": "example.png",
  "path": "C:\\Users\\...\\attachments\\image_1710000000000_abcd.png",
  "url": "file:///C:/Users/.../attachments/image_1710000000000_abcd.png",
  "createdAt": "2026-06-29T10:00:00.000Z"
}
```

支持扩展名：

- `.png`
- `.jpg`
- `.jpeg`
- `.gif`
- `.webp`
- `.bmp`

## 10. 提醒机制

提醒时间来自两种路径：

- 用户在任务卡片中手动设置 `datetime-local`
- 创建任务时由 `reminderParser.js` 的 `parseReminder(title)` 从标题中解析

当前可解析格式：

- 相对时长：`20分钟后`、`两小时后`、`1小时30分钟后`、`半小时后`
- 相对日期：`今晚八点`、`明早`、`明晚七点`、`后天下午三点半`、`三天后`
- 星期：`周五下午三点`、`本周五`、`下周一上午十点`
- 中文日期：`7月5日下午三点`、`七月六日九点`、`2026年8月1日晚上八点`
- 数字日期：`2026-07-08 16:30`、`2026/07/08 16点`
- 只有时间：`下午三点`、`晚上8:30`，已过时自动顺延到第二天

默认时间：

- 日期表达没有具体时间时默认为上午 `09:00`。
- `明早` 默认为 `09:00`，`今晚` / `明晚` 默认为 `20:00`。
- 无前缀的星期表达优先使用本周尚未过去的时间，否则顺延到下周。

调度逻辑：

1. 主进程为最近一项未提醒任务建立精确定时器。
2. 每 30 秒执行一次兜底扫描，避免单个定时器异常。
3. 系统从休眠恢复或屏幕解锁后立即补查。
4. 找出 `status !== 'done'` 且 `remindAt <= Date.now()` 的任务。
5. 写入 `notifiedAt`，并结合 `remindAt` 做跨重启去重。
6. 触发 Windows 系统通知和提示音。
7. 显示并聚焦主窗口。
8. 渲染层显示“到点提醒”面板并高亮任务。

注意：

- `notifiedTaskIds` 用于当前运行周期内快速去重，`notifiedAt` 用于跨重启去重。
- 修改提醒时间会清空 `notifiedAt` 并从内存集合移除任务 ID。
- Windows 全局通知关闭时，系统 Toast 会失败；应用仍会通过提示音、窗口唤起和应用内提醒兜底。
- 设置页的“测试提醒”会报告 Windows 通知是否被系统阻止，并提供系统通知设置入口。

## 11. 快捷键和剪贴板

默认快捷键：

- `Control+Alt+Space`：呼出快速输入
- `Control+Alt+T`：剪贴板或选区创建任务
- `Control+Alt+R`：显示 / 隐藏窗口

当 `autoCopySelection` 为 `false`：

- `Control+Alt+T` 读取当前剪贴板文本创建任务。

当 `autoCopySelection` 为 `true`：

1. 保存当前剪贴板文本和图片。
2. 通过 PowerShell `SendKeys` 模拟 `Ctrl+C`。
3. 等待约 260ms。
4. 读取新剪贴板文本。
5. 根据 `restoreClipboard` 设置恢复原剪贴板。
6. 用读取到的文本创建任务。

风险：

- 某些应用可能不响应模拟复制。
- 剪贴板里同时存在复杂格式时，目前只恢复文本或图片，不能完整恢复所有格式。
- 快捷键可能被其他应用占用，注册状态会显示在设置页。

## 12. 窗口和托盘

主窗口：

- 默认宽度 `390`
- 默认高度不超过 `680`
- 初始位置为主屏幕工作区右上角
- 无边框窗口
- `contextIsolation: true`
- `nodeIntegration: false`
- 默认不置顶，由设置控制
- 关闭窗口时默认隐藏，不退出应用

折叠行为：

- 折叠后窗口高度为 `46`
- 展开时恢复之前的窗口 bounds 或默认高度
- 折叠状态通过事件同步到渲染层

托盘：

- 单击托盘图标显示 / 隐藏窗口
- 菜单包含显示 / 隐藏、快速添加、退出

## 13. 样式和交互规范

当前 UI 是一个紧凑型工具面板，不是营销页。

维护时应遵守：

- 优先保持信息密度，不新增大面积装饰性内容。
- 控件尺寸保持稳定，避免任务数量、分组名称、按钮文字导致布局跳动。
- 不要把任务卡片再包进新的卡片容器。
- 新功能优先放在已有工具栏、设置页或任务卡片操作区。
- 字体继续使用系统中文字体栈。
- 暗色模式通过 `prefers-color-scheme: dark` 维护。
- 文案保持短句，适合窄窗口。

现有颜色变量集中在 `:root`：

- `--bg`
- `--panel`
- `--text`
- `--muted`
- `--line`
- `--accent`
- `--accent-weak`
- `--danger`
- `--warning`
- `--done`
- `--shadow`

## 14. 修改规范

### 14.1 一般规则

- 保持文件编码为 UTF-8。
- 修改文案时确认中文在 PowerShell 输出中可能需要显式设置 UTF-8。
- 优先小步修改，不做无关重构。
- 不要把 `node_modules`、`release` 里的文件当源码修改。
- 新增数据字段时，必须在 `normalizeTask()` 或对应 normalize 函数中提供兼容默认值。
- 新增设置项时，同时更新：
  - `defaultSettings()`
  - `normalizeSettings()`
  - `preload.js` 暴露的 API，如有需要
  - `renderer/app.js` 的默认设置和渲染
  - 设置页 HTML 和 CSS
  - 本文档

### 14.2 新增任务功能

如果新增任务字段，建议流程：

1. 在 `createTask()` 中写入初始值。
2. 在 `normalizeTask()` 中补默认值，保证旧数据可读。
3. 在 `updateTask()` 中处理状态联动。
4. 在 `renderTasks()` 中展示或绑定交互。
5. 如需跨进程通信，通过 preload API 暴露。
6. 更新文档中的任务结构。

### 14.3 新增 IPC

新增 IPC 时：

1. 主进程中添加 `ipcMain.handle()` 或 `webContents.send()`。
2. `preload.js` 暴露语义化方法。
3. 渲染层调用 `window.reminderApi.xxx()`。
4. 不在渲染层直接使用 `ipcRenderer`。

### 14.4 新增存储文件

新增本地存储时：

- 使用 `app.getPath('userData')` 下的固定路径。
- 文件初始化使用类似 `ensureDataFile()` 的逻辑。
- JSON 读写保持格式化输出，便于人工排查。
- 需要考虑旧版本没有该文件时的默认值。

- 可以从剪贴板图片创建任务。
- 可以给已有任务追加图片。
- 可以打开图片附件。
- 设置页保存后重启仍生效。
- 快捷键冲突时设置页能显示失败状态。
- 打包命令能生成目标产物。

## 16. 已知风险和技术债

1. 主进程文件过大  
   `main.js` 承担了过多职责，后续新增功能前最好逐步拆模块。

2. 渲染层全量重绘  
   `renderTasks()` 每次重绘都会重新生成节点和事件监听。任务量大时可能有性能问题。

3. 自动化覆盖有限
   当前只有提醒时间解析测试，调度、快捷键、剪贴板和附件仍依赖手动回归。

4. 数据无版本迁移  
   目前依赖 normalize 函数兼容旧字段，没有显式 schema version。

5. 剪贴板恢复不完整  
   只处理文本和图片，不能完整恢复富文本、多格式剪贴板内容。

6. Windows 通知依赖系统开关
   全局通知或应用通知被关闭时无法显示 Toast，目前通过提示音、窗口唤起和应用内提醒兜底。

7. 打包产物在项目目录内  
   `release/` 是构建产物，继续开发时要避免误改打包输出。

8. 数据尚无显式版本迁移
   当前依赖 `normalizeTask()` 兼容历史字段。随着归档、备份恢复和后续外部消息接入增加，应该尽快加入 `schemaVersion` 和迁移机制。

## 17. 建议后续演进

优先级较高：

- 拆分 `main.js` 的数据、提醒、窗口、快捷键模块。
- 给任务数据增加 `schemaVersion` 和迁移机制。
- 增加提醒调度和数据存储测试。
- 增加手动“检查更新/打开数据目录/打开日志”入口。

优先级中等：

- 支持窗口位置持久化。
- 支持任务备注展开编辑。
- 支持更多自然语言时间格式，例如“周五下午三点”。
- 支持重复提醒。

暂不建议优先做：

- 云同步
- 多人协作
- 移动端
- 复杂项目看板
- 权限系统
- AI 功能大改造

## 18. 接手修改时的最短阅读路径

如果只是改 UI：

1. 读 `src/renderer/index.html`
2. 读 `src/renderer/app.js`
3. 读 `src/renderer/styles.css`

如果改数据或业务逻辑：

1. 读 `src/main/main.js` 的数据读写函数
2. 读 `src/main/preload.js`
3. 读 `src/renderer/app.js` 的调用点

如果改提醒：

1. `src/main/reminderParser.js` 的 `parseReminder()`
2. `runReminderCheck()`
3. `showSystemNotification()`
4. `surfaceDueTask()`
5. 渲染层 `onReminderDue()` 和 `renderReminderPanel()`

如果改快捷键：

1. `defaultSettings()`
2. `registerShortcuts()`
3. `registerShortcut()`
4. `createClipboardTaskFromShortcut()`
5. 设置页相关 HTML 和 `renderSettings()`

## 19. 当前结论

这个工程已经完成了一个实用原型，核心价值是“快速收集 + 桌面可见 + 到点提醒”。继续迭代时，最重要的是保持轻量和低打扰，不要把它推向复杂项目管理系统。

下一阶段最适合做两类事情：

1. 补齐维护性：拆模块、加迁移、加基础测试。
2. 打磨高频体验：窗口位置、任务备注、更多时间解析、快捷键稳定性。
